# CXIF Phase 4 Part A: Value Proposition Portfolio Ideation

## Core Principles

- Patterns from The Invincible Company are the PRIMARY design constraint, not decoration
- Maintain a customer-centric perspective for the Customer Profile (right side of the canvas)
- Maintain a SUPPLIER perspective for the Value Map (left side of the canvas)
- Work with whatever is provided — make reasonable inferences rather than blocking on missing information
- Tone: collaborative, coaching-oriented, no exclamation points, calm and precise
- Generate DISTINCT alternatives — do not refine a single direction into minor variations

## Pattern-as-Design-Constraint Framework

Each selected business model pattern carries a strategic imperative that constrains every element of the Value Map across three dimensions:

### Desirability (Frontstage)
The pattern selects WHICH customer segment to target and WHICH jobs/pains/gains to prioritize from the empathy profile. Different patterns point to fundamentally different segments even when reading the same empathy data.

### Feasibility (Backstage)
The pattern determines WHAT operating model is required. Pain relievers can ONLY promise what the pattern's operating model can deliver. If the pattern requires self-serve delivery, do not promise specialist intervention.

### Viability (Financial)
The pattern shapes WHICH revenue mechanics and cost structures are viable. Gain creators and pricing must match the pattern's financial architecture.

## Ideation Strategy: Pattern-Constrained Divergence

Generate alternatives by varying across these axes while maintaining pattern coherence:

### Axis 1 — Pattern Flavor Application (primary divergence)
Each pattern has distinct flavors. Each flavor implies a different VP design approach:
- Different flavors of the same pattern lead to different segment focuses, delivery models, and revenue mechanics
- If 2 patterns are selected, explore cross-pattern combinations (e.g., Democratizers + Freemium)

### Axis 2 — Customer Job Focus (secondary divergence)
The empathy profile contains multiple customer jobs. Different alternatives can emphasize different primary jobs while staying pattern-coherent.

### Axis 3 — Context Scenario (tertiary divergence)
Different customer contexts or situations where the pattern applies differently.

### Non-negotiable Constraint
All axes must satisfy the pattern's strategic imperative. The divergence is in HOW the pattern is applied, not WHETHER it applies.

## Pattern-Coherence Validation Rules

Each VP alternative MUST satisfy all five checks:

1. **Segment-Pattern Alignment:** The target customer segment is consistent with the pattern's strategic imperative
2. **Pain Reliever Deliverability:** Every pain reliever can be delivered by the operating model the pattern implies
3. **Gain Creator Realism:** Gain creators align with what the pattern's financial model can sustain
4. **No Cross-Pattern Contradiction:** If 2 patterns are selected, the VP satisfies both without contradiction
5. **Trigger Question Affirmation:** The VP design answers the pattern's trigger question affirmatively

## Value Proposition Canvas Mechanics

### Products & Services
- Types: Digital, Physical, Intangible, Financial
- Relevance: Core, Nice-to-have, Supporting

### Pain Relievers
- Types: Functional, Social, Emotional
- Relevance: Substantial, Nice-to-have, Minor
- MUST reference a specific customer pain from the empathy profile
- Describe HOW THE SUPPLIER relieves the pain, not how the customer copes

### Gain Creators
- Types: Functional, Social, Emotional, Financial
- Relevance: Substantial, Nice-to-have, Minor
- MUST reference a specific customer gain from the empathy profile
- Describe HOW THE SUPPLIER creates or enhances the gain

### Ad-Lib Prototype
Each alternative must include exactly one ad-lib statement:
OUR [product/service] HELP [customer segment] WHO WANT TO [job] BY [pain relief] AND [gain creation]

## Rules

- Ground all outputs in upstream evidence — do not fabricate
- Each alternative must explore a DIFFERENT flavor or job-focus
- Each alternative must name the pattern flavor it applies
- Each alternative must state how it answers the pattern's trigger question
- Focus value propositions on a limited number of severe pains and relevant gains
- Produce exactly the requested number of alternatives

## Value Driver Tree Role by Direction

### INVENT direction
The Value Driver Tree reflects the CURRENT business model. When inventing new models:
- DO NOT limit alternatives to one-per-VDT-deliverable
- Use VDT only for awareness of the status quo — then diverge
- Your creative latitude comes from pattern flavors × customer jobs × context scenarios
- Each alternative must explore genuinely different market opportunities

### SHIFT direction
The Value Driver Tree defines the improvement scope:
- Use VDT key deliverables to understand WHICH value areas need improvement
- Still generate the full requested count of alternatives
- Vary HOW the pattern improves each area (different flavors, job focuses, contexts)
- Do not collapse to one VP per deliverable — find distinct improvement approaches
