# Plan: Signal Repository Integration into bmi-consultant-app

## Context
- 300 signals across 36 analyst JSON files in 5 folders (partner, product, relationship, purchase, service)
- Signals generated by standalone scripts (`bu_classify_pipeline.py` → SOC Radar 4-phase → `normalize_soc_json.py` → `generate_soc_html.py`)
- App currently runs Step 1 (SOC Radar) live from raw VoC text via LLM each time
- User wants consultants to browse pre-analyzed signals by BU and select one to start the CXIF workflow (Steps 2-9)
- App has CSV ingestion, Streamlit + React frontends, FastAPI backend, PostgreSQL persistence

## Strategy: Three-Layer Integration

### Layer 1: Signal Repository (Data Layer)
- New DB table `signal_repository` stores normalized analyst JSON signals
- Ingestion script reads all 36 `*_analyst.json` files → inserts into DB
- API endpoints to list/filter/get signals

### Layer 2: Signal-Seeded Workflow (Backend)
- New workflow entry point that accepts a pre-analyzed signal as input
- Skips Step 1a/1b (already done) → seeds state with signal data → starts at Step 2
- Reuses existing graph with conditional routing

### Layer 3: Signal Browser UI (Frontend)
- BU-grouped signal browsing view
- Signal detail panel with full 4-phase analysis
- "Start CXIF Workflow" button per signal
